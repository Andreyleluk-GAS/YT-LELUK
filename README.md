# YouTube Авто Прокси (Полное руководство по развертыванию)

Этот проект разделен на две части: **Frontend (Next.js)** и **Backend (Node.js + Xray + yt-dlp)**.
Структура репозитория позволяет хранить весь код вместе (монорепа), но разворачивать проекты бесплатно (или дешево) на разных платформах: фронтенд на Vercel, бэкенд на Koyeb или Render.

## 📦 1. Развертывание Backend (на Koyeb / Render)

Бэкенд полностью упакован в `Dockerfile` и включает в себя `Node.js`, `Python3`, `yt-dlp` и бинарный файл `Xray-core`. Он автоматически подхватывает переменную окружения `PORT`, которую предоставляют облачные провайдеры.

**Инструкция для Koyeb:**
1. Зарегистрируйтесь на [Koyeb](https://app.koyeb.com/).
2. Нажмите **Create Service** -> Выберите **GitHub**.
3. Подключите ваш репозиторий с этим проектом.
4. В разделе **Builder** выберите **Dockerfile**.
5. В поле **Override Dockerfile location** укажите путь: `backend/Dockerfile` (или убедитесь, что Koyeb сам собирает нужную папку). В качестве варианта, можно выбрать **Deploy directory**, указав `/backend`.
6. Koyeb сам подставит порт, но вы можете убедиться, что открыт стандартный `HTTP`.
7. Разверните (Deploy). Koyeb выдаст вам URL (например, `https://your-backend-app.koyeb.app`). *Сохраните его — он скоро понадобится.*

## 🖥 2. Развертывание Frontend (Vercel)

Frontend является стандартным проектом Next.js, который Vercel умеет разворачивать "из коробки".

**Инструкция для Vercel:**
1. Зарегистрируйтесь на [Vercel](https://vercel.com/) и нажмите **Add New -> Project**.
2. Подключите этот же GitHub-репозиторий.
3. В настройках проекта (раздел **Root Directory**) обязательно выберите папку `frontend` (чтобы Vercel не пытался собрать бэкенд). 
4. Раскройте секцию **Environment Variables** и добавьте новую переменную (ОБЯЗАТЕЛЬНО!):
   - **Name**: `NEXT_PUBLIC_BACKEND_URL`
   - **Value**: URL вашего бэкенда из Koyeb (например: `https://your-backend-app.koyeb.app` — **без слэша на конце!**)
5. Нажмите **Deploy**.

## 🛠 Технические особенности
- Бэкенд автоматически скачивает последнюю версию `Xray-core` при сборке контейнера.
- Прямая потоковая передача реализована через пайпинг (без сохранения на диск), что минимизирует задержки.
- Пользовательский интерфейс использует адаптивный Vanilla CSS (укрупненные элементы для браузеров Android Auto).
